# Security Policy

## Scope

This repository is primarily a specification and documentation project. Most reports will be about documentation accuracy rather than exploitable software vulnerabilities.

Use this security process for issues such as:

- credential or secret exposure
- vulnerable GitHub Actions or repository automation
- unsafe published artifacts
- private information disclosure

Do not use the security channel for normal proposal feedback, editorial fixes, or disagreements about the specification. Open a regular issue for those.

## Reporting

Report security concerns privately to the maintainers before public disclosure.

Preferred contact:

- GitHub Security Advisories, if enabled for the repository
- otherwise, contact the repository owner at `oopsyang@gmail.com`

Include:

- a short description of the issue
- impact and affected files or workflows
- reproduction steps if applicable
- any suggested mitigation

## Response Expectations

Maintainers will acknowledge receipt as quickly as practical, validate the report, and coordinate remediation before public discussion when confidentiality is necessary.
