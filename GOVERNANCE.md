# Governance

## Project Status

This repository publishes a draft proposed standard for multi-level repository navigation and deterministic routing.

The project is maintainer-led. Consensus is preferred, but maintainers make final merge and release decisions.

## Decision Model

Maintainers evaluate changes based on:

- consistency with the proposal's goals
- clarity for human readers and automation
- backward impact on adopters
- whether behavior remains deterministic and fail-closed where required

Normative changes should normally be discussed in an issue before merge when they affect conformance rules, routing semantics, or artifact requirements.

## Maintainers

Repository maintainers are the owners listed in GitHub repository settings and `CODEOWNERS`.

## Versioning

Current published draft version: `0.1.0`.

Until the proposal is declared stable, the specification is treated as draft work.

During draft status:

- section numbers are intended to remain stable when practical
- breaking normative changes may still occur
- release notes should summarize meaningful behavior changes

If formal version tags are introduced later, the tagged specification text becomes the citation target for that version.

## Translations

Translated documents are welcome, but the English canonical files are the normative source unless the repository explicitly states otherwise. Translation updates should stay aligned with the corresponding canonical source file.

## Changes To Governance

Maintainers may update this governance document as the project matures. Significant governance changes should be called out in release notes or a pinned issue.
